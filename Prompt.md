# Prompt

## 目标

- 为 `gpt-image-2` 建立按分辨率区分的路由与收费策略，而不是继续沿用“同模型统一处理、统一定价”。

## 已确认需求

- `1k`：走免费账号渠道。
- `2k`、`4k`：走 `Codex` 和 `APIMart` 渠道。
- 下游面向用户的收费调整为：
  - `1k`：`0.06` 元
  - `2k`：`0.10` 元
  - `4k`：`0.20` 元

## 涉及范围

- 当前号池 `gpt2api`：
  - `gpt-image-2` 路由选择
  - 分辨率到渠道策略映射
  - 号池侧成本/计费语义是否需要随分辨率变化
- 下游后端 `new-api`：
  - 用户扣费规则
  - 向前端暴露的价格/能力说明
- 下游前端 `new-api-web`：
  - 分辨率收费提示
  - 渠道能力或失败提示文案

## 当前边界

- 当前号池代码里的图片计费是按 `model.image_price_per_call * n` 统一计算，尚不是天然的“同模型按 resolution 分档计费”。
- 当前变更不是单点改价，而是“分辨率路由 + 分辨率收费 + 前端提示”的跨系统联动。

## 待拍板问题

- `2k/4k` 若 `Codex/APIMart` 都失败，是否允许继续回落到 `free runner`：
  - 若允许，需明确用户仍按高档收费还是按实际命中链路收费。
  - 若不允许，需明确失败提示、退款和重试策略。
